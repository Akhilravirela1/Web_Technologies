Addendum

Process

I began by outlining the core elements needed - story data structure, functions to control flow and update the UI, etc. Before writing code, I focused on plotting an imaginative narrative path with branches.

Key Tasks

  • Build initial story data and choice points
  • Implement updatePage() to populate text, choices, images dynamically
  • Handle player choices changing state
  • Create ending states with images for each branch
  • Refine story details, prose, and visuals

Creative Choices

I opted for an enchanted forest theme to evoke wonder and mystical elements. My goal was to provide a sense of exploration and unexpected outcomes from player agency.

Key Decisions

  • Fantasy setting with talking creatures and landmarks
  • Branching narrative with secrets, portals, treasures, etc.
  • Meaningful choices leading to major consequences
  • Vivid imagery complementing each story beat

Technical Implementation

Structuring the nonlinear story space with the relevance of past choices proved challenging. I overcame by iteratively building it out and console logging state.

Solutions

  • Carefully planned story data schema
  • Debugging through log statements
  • Refactoring for clarity as it evolved
